# Data Structures – Quera Course Solutions
### Author: **Amirali Sandani**

---

## 📌 Overview

This repository contains my structured and clean implementations of fundamental data structures as part of the **Quera Data Structures Course**.  
All implementations are written in **Python**, following real-world engineering standards including modular design, clean code practices, documentation, testing, and commit conventions.

The goal of this repository is to:

- Strengthen my understanding of core data structures
- Improve algorithmic thinking through hands-on implementations
- Build a professional portfolio-ready repository
- Practice writing maintainable, well‑documented engineering-grade code

---

## 📁 Repository Structure

Each topic is placed in its own dedicated directory, containing:

- Clean implementation code (`*.py`)
- Documentation or task description (`task.md`)
- Test or example code (`tests/` or inline examples)
- Optional visualizations (added later)

## 🚀 Features

- Clean, modular implementations of all classic data structures  
- Organized per-topic directory structure  
- Pythonic design using classes, type hints, and docstrings  
- Unified commit conventions (`feat`, `fix`, `refactor`, `docs`, `test`)  
- Tests or example code for every implemented algorithm

## 🧪 Tests & Examples
Every topic includes example code or testcases showing how each data structure works.

Tests are organized inside a tests/ folder for clarity and professional structure.

Examples include:

Array operations (resize, insert, delete)
Linked list traversal, insertion, deletion
Stack operations and call stack simulation
Queue implementation using singly linked list
Hash map collisions and chaining
BFS / DFS traversal for graphs
Binary search tree operations (insert, search, delete)

---

## 📜 Code Style & Conventions
This repository follows standard engineering practices:

Commit Messages:
text
feat: add binary search tree insert logic
fix: correct boundary condition in queue dequeue
refactor: improve linked list traversal
docs: add explanation for BFS
test: add unit tests for stack implementation

Formatting:
Auto-format using Black:
bash
pip install black
black .
